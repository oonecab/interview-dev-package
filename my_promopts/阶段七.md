2. 设备模块边界状态
检查并补齐：
- 楼栋加载失败：已有 Alert，可确认文案友好。
- 设备列表加载失败：显示错误和可重试入口，不能只空表格。
- 当前楼栋无设备：显示“当前楼栋暂无设备”。
- 当前筛选无设备：显示“当前筛选条件下暂无设备”，最好能区分无设备和筛选无结果。
- 设备详情不存在或加载失败：抽屉中显示错误，并允许关闭。
- 设备详情加载中：保持 loading。
- 设备统计在全量设备加载失败时要有合理降级，不要静默显示全 0 造成误导。

3. 告警面板边界状态

4. 工单模块边界状态
检查并补齐：
- 工单列表加载失败：显示错误和可重试入口。
- 当前无工单：显示“暂无工单”。
- 当前状态筛选无工单：显示“当前筛选条件下暂无工单”。
- 创建工单失败：已有 message，确认错误来自服务端 message 时能展示。
- 创建工单提交中：按钮 loading，避免重复提交。
- 设备下拉加载失败或无设备时，要有可理解状态。
- 工单状态流转失败：保留抽屉，不要关闭；显示失败提示。
- 工单状态流转成功：刷新列表，给成功提示。
- 已完成工单不能继续推进，已有提示请保持。

5. AI 聊天与 Tool Calling 边界状态
检查并补齐：
- 普通 /api/chat 失败：已有错误消息，确认文案友好。
- 空输入禁发，Enter 发送，Shift+Enter 换行保持不变。
- Tool arguments JSON 解析失败：已有失败状态，确认不会继续假成功。
- Tool API 调用失败：工具卡片失败，最终用户可见。
- 未知工具名：工具卡片失败，用户可见。
- 多轮 tool_calls 达到 MAX_TOOL_ROUNDS：提示用户更具体提问。
- create_work_order tool 成功后刷新工单列表，失败不刷新。
- 消息较长时不撑破右侧栏。
- 关闭/展开 AI 助手后历史保留。
