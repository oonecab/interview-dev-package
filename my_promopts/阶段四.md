请基于当前项目继续开发 milestone/development-milestones.md 的阶段 4：AI 助手基础聊天。先阅读 question.md、attachments/wireframes.md、attachments/api-spec.md 和现有 src 代码，保持当前 React + TypeScript + Ant Design + React Query 风格。

目标：
实现右侧 AI 助手的基础聊天 UI 和普通文本回复。阶段 4 只做普通聊天，不实现 Tool Calling 闭环；如果 /api/chat 返回 tool_calls，可以先展示“工具调用将在下一阶段支持”的友好提示，真正的工具执行留到阶段 5。

具体要求：

1. 替换 AppShell 右侧 AI 助手占位
- 当前 src/components/layout/AppShell.tsx 中右侧栏只有“AI 助手功能开发中...”。
- 新增聊天组件，建议放在 src/components/chat/ChatSidebar.tsx。
- 保留现有顶部“AI 助手/关闭助手”按钮和右侧栏展开/收起行为。
- 关闭后再次展开时应保留本轮对话历史，不要因为组件卸载导致消息清空。可以把消息状态提升到 AppShell，或用其他简单可靠方式实现。

2. 聊天 UI
- 右侧栏宽度沿用现有 360px。
- 顶部显示“AI 助手”和关闭按钮。
- 消息区展示：
  - 欢迎消息，内容参考线框图：可以帮你查询设备状态、查看告警信息、创建维修工单。
  - 用户消息靠右。
  - assistant 消息靠左。
- 底部输入区包含 TextArea/Input 和发送按钮。
- 支持 Enter 发送，Shift+Enter 换行。
- 空输入不能发送，发送按钮 disabled。
- 发送中展示 loading 状态，并避免重复发送。
- 新消息到达后自动滚动到底部。
- 接口失败时在消息区或 Ant Design message 中给出友好错误提示，不能白屏或静默失败。

3. 接入 Chat API
- 使用已有 src/api/chat.ts 的 sendChat(messages)。
- 使用已有 src/types/chat.ts 类型；如果类型不够，做最小必要扩展。
- 请求体格式必须符合 attachments/api-spec.md：
  { messages: [{ role: 'user', content: '...' }, ...] }
- 普通文本响应：
  { role: 'assistant', content: '...' }
  要追加到消息列表展示。
- 如果响应 content 为 null 且包含 tool_calls，本阶段不要执行工具，只追加一条 assistant 提示，例如“我识别到需要调用工具，工具调用将在下一阶段接入。”

4. 样式和体验
- 使用 Ant Design 组件即可，不要引入新的 UI 库。
- 视觉保持管理后台风格，紧凑、清晰、可扫描。
- 消息气泡不要过宽，长文本要换行。
- 右侧栏高度内自适应，底部输入区固定在底部，消息区可滚动。

5. 测试
- 增加至少一个聊天相关测试，使用 mock fetch 或 mock sendChat，不依赖真实 mock server。
- 建议覆盖：
  - 输入文本后点击发送，会展示用户消息和 assistant 回复；
  - 空输入不会触发发送；
  - 接口失败会展示错误提示。
- 保持已有测试通过。

6. 顺手小整理
- 如改动设备 query key，最多做小范围整理：可把设备详情 queryKey 从 ['deviceDetail', deviceId] 调整为 ['devices', 'detail', deviceId]，但不要扩大重构范围。

验收：
- npm run test 必须通过。
- npm run build 必须通过。
- 不要实现阶段 5 的 query_devices/query_alerts/create_work_order 工具执行器。
- 最后总结新增/修改的文件、实现了哪些阶段 4 功能、测试和构建结果。
