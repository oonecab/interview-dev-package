必须修复的 3 个 bug：
1. fetchAlerts 缺少 buildingId 依赖
- 原组件 useCallback 依赖数组为空，导致 buildingId 变化后仍请求旧楼栋。
- 修复后 buildingId 变化必须重新拉取告警。

2. 自动刷新定时器没有清理
- 原组件 setInterval 后没有 clearInterval，且 effect 依赖不完整。
- 修复后关闭自动刷新、组件卸载、buildingId/fetchAlerts 变化时都必须清理旧 timer。

3. 确认告警时直接修改 React state
- 原组件 alert.acknowledged = true 后 setAlerts(alerts)，属于直接 mutation，可能不触发正确渲染。
- 修复后必须使用不可变更新，例如 setAlerts(prev => prev.map(...))。

2. 功能要求
DeviceAlertPanel props 至少包含：
- buildingId: string
- onAlertClick?: (alert: Alert) => void

组件功能：
- 根据 buildingId 拉取 /api/alerts?buildingId=...
- 展示当前楼栋告警列表。
- 支持告警级别筛选：
  - 全部
  - 严重 critical
  - 警告 warning
  - 信息 info
- 支持未确认筛选。
  - 可以做成 Select 选项 “未确认”，也可以做成 Checkbox “仅未确认”。
- 支持自动刷新开关。

- 支持确认告警：
  - 点击“确认”调用 POST /api/alerts/:id/ack。
  - 成功后本地状态不可变更新 acknowledged。
  - 失败时给出友好提示。
- 支持空状态：
  - 无告警时显示“暂无告警”。
- 支持加载和错误状态：
  - 加载中不要白屏。
  - 请求失败要显示 Alert 或 message。

4. 数据刷新和 React Query
可选实现方式：
- 可以用 React Query 实现 alerts 查询和 ack mutation；
- 也可以用 useEffect/useState 实现，但必须正确处理依赖和清理 timer。
推荐使用 React Query + mutation，但如果使用 interval，仍要明确清理。
注意：
- 如果使用 React Query refetchInterval，也要确保 autoRefresh=false 时停止刷新。
- 确认告警后可以：
  - 乐观更新 query cache；
  - 或本地不可变更新；
  - 或 invalidate alerts query。
但请保证 UI 能立即/正确反映已确认状态。
