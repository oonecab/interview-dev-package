## 本阶段功能范围

实现完整工单管理模块，包括：

1. 顶部导航切换
   - 当前 App 只有设备看板
   - 请在顶部导航或主区域增加页面切换：
     - 设备看板
     - 工单管理
   - 切换到“设备看板”时显示现有 `DeviceDashboard`
   - 切换到“工单管理”时显示新的工单管理页面
   - 不要破坏现有设备看板功能

2. 工单列表
   - 调用 `getWorkOrders`
   - 使用 Ant Design Table 展示：
     - 工单编号
     - 标题
     - 状态
     - 优先级
     - 关联设备
     - 创建时间
   - 支持 loading、empty、error 状态
   - 点击工单行打开详情 Drawer

3. 工单状态筛选
   - 支持按状态筛选：
     - 全部
     - 待派单 `pending`
     - 已派单 `assigned`
     - 处理中 `in_progress`
     - 已完成 `completed`
   - 筛选时调用 `getWorkOrders({ status })`
   - “全部”状态不传 status 参数

4. 创建工单
   - 页面右上角增加“创建工单”按钮
   - 点击后打开 Modal 或 Drawer 表单
   - 表单字段：
     - 标题：必填
     - 描述：必填
     - 关联设备：必填，下拉选择设备
     - 优先级：必填，支持 高/中/低
   - 关联设备下拉可以调用 `getDevices()` 获取全部设备
   - 提交时调用 `createWorkOrder`
   - 创建成功后：
     - 关闭表单
     - 清空表单
     - 刷新工单列表
     - 显示成功提示
   - 创建失败时显示友好错误提示，不要白屏

5. 工单详情
   - 点击工单行打开详情 Drawer
   - 展示：
     - 编号
     - 标题
     - 状态
     - 优先级
     - 关联设备
     - 创建时间
     - 更新时间
     - 描述
   - 展示状态流转进度：
     - 待派单 → 已派单 → 处理中 → 已完成
   - 可以使用 Ant Design Steps

6. 状态流转
   - 在工单详情里提供“推进到下一状态”按钮
   - 状态流转规则：
     - `pending` → `assigned`
     - `assigned` → `in_progress`
     - `in_progress` → `completed`
     - `completed` 没有下一状态，按钮禁用
   - 点击后调用 `updateWorkOrderStatus`
   - 成功后：
     - 刷新工单列表
     - 详情中显示最新状态
     - 显示成功提示
   - 失败时显示错误提示，尤其是 422 非法状态流转

7. 从设备详情进入创建工单
   - 当前 `DeviceDetailDrawer` 中有“创建工单（功能开发中）”按钮
   - 本阶段可以选择以下较简单方案之一：
     - 方案 A：先保持 disabled，不处理
     - 方案 B：打通入口，点击后切换到工单管理并打开创建工单弹窗，预填设备
   - 如果实现方案 B，注意不要把状态传递搞复杂
   - 如果时间不够，优先完成工单管理主流程，设备详情入口可以保留 TODO
