## 本阶段功能范围

实现模块 A：设备状态看板。

必须包含：

1. 整体布局
   - 顶部导航栏：显示“星汇智慧空间”
   - 左侧边栏：楼栋选择器 + 设备状态筛选
   - 主内容区：设备统计概览 + 设备列表
   - 右侧 AI 助手区域可以先做占位，不实现聊天逻辑
   - 最小支持宽度按 1280px 桌面端设计即可

2. 楼栋选择器
   - 调用 `getBuildings`
   - 默认选中第一个楼栋
   - 点击楼栋后刷新设备列表
   - 展示楼栋名称和设备数量

3. 设备筛选
   - 支持状态筛选：
     - 全部
     - 正常 `normal`
     - 告警 `warning`
     - 故障 `fault`
     - 离线 `offline`
   - 支持设备类型筛选：
     - 全部
     - 电梯 `elevator`
     - 空调 `hvac`
     - 水泵 `pump`
     - 照明 `lighting`
     - 消防 `fire_pressure`

4. 设备列表
   - 调用 `getDevices({ buildingId, status, type })`
   - 使用 Ant Design Table 展示：
     - 设备名称
     - 楼栋/楼层
     - 类型
     - 状态
     - 最后更新时间
   - 状态用 Tag 展示，并使用清晰颜色：
     - 正常：绿色
     - 告警：橙色
     - 故障：红色
     - 离线：灰色
   - loading、empty、error 状态都要有友好展示

5. 统计概览
   - 基于当前筛选后的设备列表统计
   - 展示各设备类型数量
   - 展示各设备状态数量
   - 使用 Ant Design Card / Statistic / Tag 等组件即可
   - 不要做营销式大屏，保持后台管理系统风格，清爽、可扫描

6. 设备详情
   - 点击设备行后打开右侧 Drawer 或 Modal
   - 调用 `getDeviceDetail(id)`
   - 展示：
     - 设备名称
     - 类型
     - 楼栋/楼层
     - 状态
     - 最后更新时间
     - 最近告警记录
   - 告警记录展示：
     - 级别
     - 消息
     - 时间
     - 是否已确认
   - Drawer 内可以放一个“创建工单”按钮，但本阶段只做 disabled 或占位提示，不实现创建工单

7. 告警类型补充
   - 检查 `src/types/domain.ts`
   - mock server 的 alert 数据包含 `buildingId`
   - 如果当前 `Alert` 类型缺少 `buildingId`，请补上
   - 确保不破坏现有 build/test
